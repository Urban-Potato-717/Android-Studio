# BookLog 발표 코드 이해 치트시트

목표는 코드를 외우는 것이 아니라, 시연하면서 "이 화면은 어떤 파일이 담당하고, 데이터가 어디서 와서 어떻게 저장되는지"를 말할 수 있게 되는 것이다.

## 0. 한 문장 요약

BookLog는 서버 없이 Android 앱 내부 SQLite DB에 회원, 책, 리뷰를 저장하고, 카카오 도서 검색 API로 외부 책 정보를 가져와 리뷰를 남길 수 있는 도서 리뷰 앱이다.

## 1. 전체 구조 먼저 잡기

앱은 크게 4개 층으로 보면 된다.

```text
화면 Activity
  LoginActivity, HomeActivity, SearchActivity, BookDetailActivity ...

화면에 목록을 그리는 Adapter
  BookAdapter, ReviewAdapter, MyReviewAdapter

데이터 저장/조회
  DBHelper(SQLite)

보조 기능
  Session, KakaoApi, Covers, ImageLoader, BottomNav
```

발표할 때는 이렇게 말하면 된다.

> "이 프로젝트는 Activity가 화면 하나를 담당하고, DBHelper가 SQLite 데이터를 관리합니다. 책이나 리뷰처럼 반복되는 목록은 Adapter가 XML 레이아웃에 데이터를 연결해서 보여주는 구조입니다."

## 2. 앱 시작 흐름

열어볼 코드:

- `app/src/main/AndroidManifest.xml`
- `LoginActivity.java`
- `Session.java`

흐름:

```text
앱 실행
  -> AndroidManifest.xml에서 MAIN/LAUNCHER로 등록된 LoginActivity 실행
  
  <activity
            android:name=".LoginActivity"
            android:exported="true">
            <intent-filter>
                <action android:name="android.intent.action.MAIN" /> 여기 메인
                <category android:name="android.intent.category.LAUNCHER" /> 폰 키면 실행 (런처)
            </intent-filter>
        </activity>

  -> LoginActivity에서 세션 확인
  -> 이미 로그인되어 있으면 HomeActivity로 이동
  -> 아니면 로그인 화면 표시
```

발표 멘트:

> "앱의 시작 화면은 Manifest에 등록된 LoginActivity입니다. LoginActivity에서는 먼저 SharedPreferences에 로그인 세션이 있는지 확인하고, 있으면 바로 홈으로 보내고 없으면 로그인 화면을 보여줍니다."

핵심 코드:

```java
if (Session.isLoggedIn(this)) {
    goHome();
    return;
}
```

이해 포인트:

- `Activity`: Android에서 화면 하나를 의미한다.
- `Intent`: 다른 화면으로 이동할 때 사용하는 객체다.
- `SharedPreferences`: 간단한 로그인 상태 같은 값을 앱 내부에 저장하는 기능이다.

## 3. 로그인/회원가입

열어볼 코드:

- `LoginActivity.java`
- `SignupActivity.java`
- `DBHelper.java`
- `Pw.java`
- `Session.java`

로그인 흐름:

```text
사용자가 아이디/비밀번호 입력
  -> LoginActivity에서 빈 값 검사
  -> DBHelper.login(username, password)
  -> DB에서 사용자 조회
  -> 성공하면 Session.login()
  -> HomeActivity 이동
```

회원가입 흐름:

```text
사용자가 아이디/닉네임/비밀번호 입력
  -> SignupActivity에서 빈 값, 비밀번호 확인 검사
  -> DBHelper.signup()
  -> users 테이블에 저장
  -> 완료 후 로그인 화면으로 돌아감
```

발표 멘트:

> "로그인과 회원가입은 서버가 아니라 SQLite에 저장된 users 테이블을 사용합니다. 비밀번호는 그대로 저장하지 않고 Pw 클래스에서 해시 처리해서 저장하도록 했습니다."

질문받을 수 있는 것:

Q. 실제 서비스에서도 이렇게 하나요?

> "실제 서비스라면 서버 인증과 토큰을 써야 합니다. 이 프로젝트는 수업용 데모라서 로컬 SQLite와 SharedPreferences로 로그인 흐름을 구현했습니다."

## 4. 홈 화면과 인기 도서

열어볼 코드:

- `HomeActivity.java`
- `activity_home.xml`
- `DBHelper.java`
- `BookAdapter.java`
- `list_item_book.xml`

홈 화면 흐름:

```text
HomeActivity 실행
  -> activity_home.xml 화면 연결
  -> DBHelper.getPopularBooks()로 인기 도서 조회
  -> BookAdapter에 책 목록 전달
  -> ListView에 표시
```

핵심 코드:

```java
adapter = new BookAdapter(this, DBHelper.get(this).getPopularBooks());
listBooks.setAdapter(adapter);
```

인기 도서 기준:

```text
base_count + 실제 reviews 테이블 리뷰 수
```

DBHelper에서는 이 값이 많은 순서로 정렬한다.

```sql
ORDER BY total_count DESC
```

발표 멘트:

> "홈 화면의 인기 도서는 DBHelper에서 가져옵니다. 여기서 인기 기준은 별점순이 아니라 기본 커뮤니티 리뷰 수와 실제 작성된 리뷰 수를 더한 total_count가 많은 순서입니다."

## 5. 홈 검색과 카카오 검색 차이

이 부분은 질문받기 좋으므로 꼭 구분해서 말한다.

### 5.1 홈에서 글자를 입력할 때

열어볼 코드:

- `HomeActivity.java`
- `DBHelper.searchBooks()`

흐름:

```text
검색창에 글자 입력
  -> TextWatcher 동작
  -> refreshList(query)
  -> query가 비어 있으면 인기 도서
  -> query가 있으면 DBHelper.searchBooks(query)
  -> 로컬 DB의 제목 LIKE 검색 결과 표시
```

핵심 코드:

```java
etSearch.addTextChangedListener(new TextWatcher() {
    @Override public void onTextChanged(CharSequence s, int a, int b, int c) {
        refreshList(s.toString().trim());
    }
});
```

발표 멘트:

> "홈 검색창은 입력하는 즉시 로컬 DB 안의 책 제목을 필터링합니다. 그래서 이미 앱에 저장되어 있는 책만 바로 걸러서 보여줍니다."

### 5.2 키보드 검색 버튼을 눌렀을 때

열어볼 코드:

- `HomeActivity.java`
- `SearchActivity.java`
- `KakaoApi.java`

흐름:

```text
키보드 검색 버튼 클릭
  -> HomeActivity에서 SearchActivity로 이동
  -> 검색어를 Intent extra로 전달
  -> SearchActivity에서 doSearch(query)
  -> KakaoApi.search(query)
  -> 카카오 API 결과를 BookAdapter로 표시
```

핵심 코드:

```java
Intent intent = new Intent(this, SearchActivity.class);
intent.putExtra("query", q);
startActivity(intent);
```

발표 멘트:

> "키보드의 검색 버튼을 누르면 로컬 필터링이 아니라 SearchActivity로 넘어갑니다. 이 화면에서는 카카오 도서 검색 API를 호출해서 앱 DB에 없던 외부 책도 검색할 수 있습니다."

차이 정리:

```text
홈 입력 중 검색 = 로컬 DB 제목 필터링
검색 버튼 실행 = 카카오 API 외부 검색
```

## 6. 검색 결과를 누르면 어떻게 되나

열어볼 코드:

- `SearchActivity.java`
- `DBHelper.insertOrGetBook()`
- `BookDetailActivity.java`

흐름:

```text
검색 결과 책 클릭
  -> SearchActivity의 OnItemClickListener 실행
  -> DBHelper.insertOrGetBook()
  -> 같은 제목+저자가 있으면 기존 book_id 사용
  -> 없으면 books 테이블에 새로 저장
  -> BookDetailActivity로 book_id 전달
```

핵심 코드:

```java
long bookId = DBHelper.get(this).insertOrGetBook(...);
intent.putExtra("book_id", bookId);
startActivity(intent);
```

발표 멘트:

> "검색 결과를 클릭하면 바로 상세 화면으로 가는 것이 아니라, 먼저 SQLite books 테이블에 저장하거나 기존 책을 재사용합니다. 그 다음 book_id를 넘겨서 상세 화면을 엽니다."

## 11. 평균 별점 계산

열어볼 코드:

- `DBHelper.java`
- `Book.java`

계산 방식:

```text
리뷰 수 = base_count + 실제 리뷰 수
별점 합 = base_sum + 실제 리뷰 별점 합
평균 별점 = 별점 합 / 리뷰 수
```

왜 이렇게 했나:

> "데모 앱이라 처음부터 화면이 비어 보이지 않게 기본 커뮤니티 리뷰 수와 별점 합을 시드 데이터로 넣어두었습니다. 사용자가 작성한 리뷰는 reviews 테이블에 추가되고, 조회할 때 기본 값과 실제 리뷰를 합쳐 평균을 계산합니다."

핵심 코드 개념:

```sql
LEFT JOIN reviews r ON r.book_id = b.book_id
GROUP BY b.book_id
```

이 뜻:

- 책을 기준으로 리뷰를 붙인다.
- 리뷰가 없는 책도 보여야 하므로 `LEFT JOIN`을 쓴다.
- 책별로 리뷰 수와 별점 합을 모으기 위해 `GROUP BY`를 쓴다.

## 15. 시연 순서 추천

시연은 기능이 자연스럽게 연결되도록 아래 순서로 한다.

```text
1. 로그인 화면
2. demo / demo 로그인
3. 홈 화면 인기 도서 설명
4. 검색창에 "소년" 입력해서 로컬 필터링 보여주기
5. 검색어 입력 후 키보드 검색으로 카카오 검색 화면 이동
6. 검색 결과 하나 클릭해서 상세 화면 이동
7. 상세 화면에서 평균 별점, 리뷰 목록, 정렬 보여주기
8. 스포일러 리뷰 클릭해서 경고 다이얼로그 보여주기
9. 리뷰 작성 화면에서 별점/한줄평 저장
10. 상세 화면으로 돌아와 리뷰 수 갱신 확인
11. 내 리뷰 탭에서 방금 쓴 리뷰 확인
12. 마이페이지에서 리뷰 수와 로그아웃 보여주기
```

## 16. 시연 대본

### 시작

> "제가 만든 앱은 BookLog라는 도서 리뷰 앱입니다. 사용자는 책을 검색하고, 책 상세 화면에서 별점과 한줄평을 남기고, 다른 리뷰와 평균 별점을 확인할 수 있습니다. 서버는 따로 두지 않고 SQLite로 로컬 데이터를 관리했습니다."

### 로그인

> "처음 실행하면 LoginActivity가 열립니다. 이미 로그인된 상태라면 SharedPreferences의 Session 값을 확인해서 바로 홈으로 이동하고, 아니면 로그인 화면을 보여줍니다. 데모 계정은 demo/demo입니다."

### 홈

> "홈 화면은 HomeActivity입니다. 처음에는 DBHelper의 getPopularBooks로 책 목록을 가져오고, BookAdapter가 ListView에 책 카드를 표시합니다. 인기 도서 기준은 기본 리뷰 수와 실제 리뷰 수를 합친 total_count가 많은 순서입니다."

### 홈 검색

> "검색창에 입력하면 TextWatcher가 동작해서 refreshList를 호출합니다. 이때 검색어가 비어 있으면 인기 도서를 보여주고, 검색어가 있으면 DBHelper.searchBooks로 로컬 DB의 제목을 검색합니다."

### 카카오 검색

> "키보드 검색 버튼을 누르면 SearchActivity로 넘어갑니다. 여기서는 로컬 검색이 아니라 KakaoApi 클래스에서 카카오 도서 검색 API를 호출합니다. 결과는 Book 객체 리스트로 바꿔서 같은 BookAdapter로 보여줍니다."

### 상세

> "책을 누르면 book_id를 Intent로 넘겨 BookDetailActivity를 엽니다. 상세 화면은 이 book_id로 DB에서 책 정보와 리뷰 목록을 다시 조회합니다. 그래서 홈에서 들어오든 검색 결과에서 들어오든 같은 상세 화면을 재사용합니다."

### 리뷰 정렬

> "정렬 버튼을 누르면 AlertDialog로 최신순과 별점순을 선택할 수 있습니다. 선택값에 따라 DBHelper.getReviewsForBook의 정렬 조건이 바뀌고 리뷰 목록을 다시 불러옵니다."

### 스포일러

> "스포일러가 체크된 리뷰는 ReviewAdapter에서 내용을 바로 보여주지 않고 가립니다. 사용자가 탭하면 확인 다이얼로그를 띄우고, 확인하면 revealed 값을 바꿔서 해당 리뷰 내용을 보여줍니다."

### 리뷰 작성

> "리뷰 작성 화면에서는 별점, 한줄평, 스포일러 여부를 입력합니다. 저장하면 reviews 테이블에 추가되고, 상세 화면으로 돌아왔을 때 onResume에서 다시 조회해서 평균 별점과 리뷰 수가 갱신됩니다."

### 내 리뷰 / 마이페이지

> "내 리뷰 탭은 Session에 저장된 현재 user_id 기준으로 내가 작성한 리뷰만 가져옵니다. 마이페이지도 같은 Session 정보를 사용해서 닉네임, 아이디, 리뷰 수를 보여주고 로그아웃 시 세션을 지웁니다."

## 17. 코드 열어볼 순서

발표 중 코드를 열어야 한다면 이 순서가 가장 안전하다.

```text
1. AndroidManifest.xml
   앱 시작점이 LoginActivity라는 것만 보여주기

2. LoginActivity.java
   세션 확인, 로그인 성공 후 홈 이동

3. HomeActivity.java
   인기 도서 조회, 검색창 TextWatcher, SearchActivity 이동

4. DBHelper.java
   테이블 구조, getPopularBooks, searchBooks, 평균 별점 계산

5. BookAdapter.java
   Book 데이터를 list_item_book.xml에 연결

6. SearchActivity.java
   카카오 검색 실행, 결과 클릭 시 insertOrGetBook

7. KakaoApi.java
   카카오 API 호출, JSON을 Book 리스트로 파싱

8. BookDetailActivity.java
   book_id로 책/리뷰 조회, 정렬, 리뷰 작성 이동

9. ReviewAdapter.java
   리뷰 표시, 스포일러 블라인드 처리

10. WriteReviewActivity.java
    리뷰 저장
```

전부 열 필요는 없다. 보통은 `HomeActivity`, `DBHelper`, `SearchActivity`, `BookDetailActivity`, `WriteReviewActivity` 정도만 열어도 충분하다.

## 18. 모르면 이렇게 말하기

Q. 왜 RecyclerView가 아니라 ListView를 썼나요?

> "수업에서 다룬 기본 목록 구조를 중심으로 구현하려고 ListView와 BaseAdapter를 사용했습니다. 실제 서비스 수준에서는 RecyclerView가 더 일반적이지만, 이 프로젝트에서는 기능 흐름을 보여주는 데 ListView로 충분하다고 판단했습니다."

Q. 왜 서버를 안 썼나요?

> "모바일 프로그래밍 프로젝트 범위에서 앱 내부 동작을 완성하는 것이 목표였고, 서버 없이도 회원, 책, 리뷰 저장 흐름을 보여줄 수 있도록 SQLite를 사용했습니다."

Q. 카카오 API 키가 없으면 어떻게 되나요?

> "카카오 검색과 표지 보강은 동작하지 않고 안내 메시지를 보여줍니다. 하지만 로그인, 홈 인기 도서, 상세, 리뷰 작성 같은 로컬 기능은 SQLite 기반이라 계속 동작합니다."

Q. 평균 별점은 어디서 계산되나요?

> "DBHelper의 공통 책 조회 쿼리에서 계산됩니다. books 테이블의 base_count/base_sum과 reviews 테이블의 실제 리뷰 수/별점 합을 합쳐서 평균을 냅니다."

Q. 검색은 DB 검색인가요, API 검색인가요?

> "둘 다 있습니다. 홈에서 글자를 입력하는 동안은 로컬 DB 제목 검색이고, 키보드 검색 버튼을 누르면 SearchActivity에서 카카오 API 검색을 합니다."

Q. 리뷰 작성 후 화면이 어떻게 갱신되나요?

> "리뷰 작성 화면에서 저장 후 finish로 상세 화면에 돌아옵니다. BookDetailActivity의 onResume에서 책 정보와 리뷰 목록을 다시 불러오기 때문에 새 리뷰와 평균 별점이 반영됩니다."

Q. 스포일러 공개 상태는 저장되나요?

> "아니요. revealed 값은 현재 화면에서만 쓰는 임시 값입니다. DB에는 리뷰가 스포일러인지 여부만 저장됩니다."

## 19. Java/Android 최소 개념만

발표에 필요한 개념만 알면 된다.

```text
class
  관련 코드 묶음. 예: HomeActivity 클래스는 홈 화면 코드 묶음.

method
  함수. 예: onCreate(), refreshList(), doSearch().

onCreate()
  Activity가 처음 만들어질 때 실행되는 함수.

onResume()
  화면이 다시 보일 때 실행되는 함수. 리뷰 저장 후 갱신에 사용.

Intent
  화면 이동과 데이터 전달에 사용.

findViewById()
  XML에 있는 버튼, 텍스트, 리스트를 Java 코드에서 가져오는 함수.

setOnClickListener()
  사용자가 클릭했을 때 실행할 코드를 등록.

Adapter
  데이터 리스트를 화면 리스트 아이템으로 바꿔주는 연결 장치.

SQLiteOpenHelper
  Android 내부 SQLite DB 생성과 업그레이드를 관리하는 클래스.

ExecutorService
  네트워크처럼 오래 걸리는 작업을 백그라운드에서 실행.

Handler
  백그라운드 작업 결과를 다시 화면 UI에 반영할 때 사용.
```

## 21. 정말 짧은 30초 요약

> "BookLog는 Android Java로 만든 도서 리뷰 앱입니다. Activity별로 화면을 나누고, SQLiteOpenHelper인 DBHelper가 회원, 책, 리뷰 데이터를 관리합니다. 홈에서는 DB에 저장된 책을 인기순으로 보여주고, 검색창 입력은 로컬 제목 검색으로 필터링합니다. 검색 실행을 누르면 SearchActivity에서 카카오 도서 API를 호출해 외부 책을 검색합니다. 책 상세 화면은 book_id를 기준으로 책 정보와 리뷰를 조회하고, 리뷰 작성 후에는 reviews 테이블에 저장되어 평균 별점과 리뷰 수가 다시 계산됩니다."
